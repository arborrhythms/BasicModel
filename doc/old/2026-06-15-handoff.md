## Recorded diagnosis (no pending code change)

**Why MM_20M fails XOR** (resolved; see `mm-configs-mps-xor-debugging.md`): not
butterfly — `lexer=byte` routes XOR down the AR byte cursor
(`use_byte_cursor=True`), whose target is the input bytes, so the `{0,1,1,0}`
labels are replaced by zero stubs (`outputTensor=byte_stub_output`) and the head
never trains on XOR → flat 0.5. Ablations rule out VQ/L1/reconScale/LR/butterfly.
MM_20M as written is a self-supervised byte autoencoder, not XOR-supervised; the
cheapest fix-test is `WholeSpace <lexer>byte</lexer>` → `<lexer>word</lexer>`.
